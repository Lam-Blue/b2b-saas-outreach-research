# Expert Selection Workspace

## Purpose

This workspace holds evidence and working notes for selecting public experts for the B2B SaaS cold outreach research project.

`research/sources.md` remains frozen until explicit human approval for Task 07. It is the source of truth for research areas, selection criteria, and final publishing structure.

## Working Files and Folders

- `research-log.md`: Candidate-level progress dashboard and provisional working status.
- `source-library.md`: Searchable index of retained source records.
- `candidate-dossiers/`: One folder per researched candidate, with a dossier and source records.
- `discovery/coverage-gap-brief.md`: Current coverage map and discovery briefs.
- `discovery/external-candidate-leads.md`: Lightweight external discovery log.
- `selection/selection-matrix.md`: Comparison table for deep-screened candidates.
- `selection/expert-selection-review.md`: Human-review recommendation package.

Source records must be reusable for later insight retrieval. Capture only the relevant excerpt, timestamp, metadata, retrieval note, and uncertainty needed to verify conclusions.

## Task Sequence

1. Initialize the expert research workspace.
2. Research the initial candidate list.
3. Map current coverage and define discovery gaps.
4. Discover external candidate leads.
5. Deep-screen external candidates.
6. Build the expert selection package.
7. Publish approved selection only after explicit human approval.
