# Screen B2B Outreach Experts

## Purpose

Evaluate assigned expert candidates using the criteria already defined in `research/sources.md`.

This skill verifies evidence and updates candidate-facing research output. It does not redefine the criteria or invent new decision labels.

## Required reading

Before starting:

1. Read all files in `.agents/rules/`.
2. Read `research/sources.md`.
3. Read the active task plan.
4. Confirm the assigned candidate list and allowed output files.

Do not research beyond the assigned batch.

## Workflow

### 1. Read the candidate context

For each assigned candidate, identify:

* the current claimed area
* the specific criteria that still need proof
* the evidence needed to verify or challenge that claim
* the required decision field or status in `research/sources.md`

### 2. Gather verification evidence

Prioritize evidence in this order:

1. Candidate’s company, website, case study, or first-party work
2. Original LinkedIn content
3. Original video, podcast, webinar, or newsletter material
4. Credible third-party interview with direct statements from the candidate

Look specifically for:

* direct operating experience
* documented outbound practice
* clear B2B SaaS relevance
* recent original material
* evidence for a narrow, specific expert area

### 3. Store evidence first

Save useful evidence following `.agents/rules/02-source-capture.md`.

Do not update the candidate-review output until supporting evidence is stored.

### 4. Evaluate against the existing criteria

Use the exact fields, scoring logic, and labels already present in `research/sources.md`.

When evidence is incomplete:

* preserve the unknown
* use the existing uncertainty label
* state what is missing
* do not compensate with reputation or generic content

### 5. Update concise candidate output

Keep the candidate-facing update short.

Include only:

* proven operating area
* direct evidence of practice
* relevance to the project
* material availability or recency where required
* existing decision or review status

Raw excerpts belong in evidence files, not in the candidate table.

## Quality gate

Before completing each candidate, verify:

* Every meaningful claim has a stored source.
* The candidate’s area is specific and evidence-supported.
* Practitioner credibility is demonstrated, not assumed.
* Any missing proof is visible.
* No criteria or decision labels were changed.
* No unassigned candidate was edited.
* No next batch was started.

## Final response

Use the repository’s standard completion report.
