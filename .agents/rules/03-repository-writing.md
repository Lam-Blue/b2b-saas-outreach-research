Repository Writing and Editing
Source of truth

research/sources.md is the source of truth for:

expert-selection criteria
candidate-review fields
decision labels
target research areas
concise candidate-facing summaries

Do not create a second criteria table elsewhere.

File responsibilities

research/sources.md

Holds concise candidate-facing information.
Focuses on proven area, evidence-backed relevance, and review status.
Must not become a raw-evidence archive or long-form biography.

research/linkedin-posts/

Holds source records from original LinkedIn material.

research/youtube-transcripts/

Holds timestamped evidence extracts and notes from videos or podcasts.

research/other/

Holds source records from websites, newsletters, company pages, interviews, case studies, and similar material.

.agents/task-plans/

Defines current scope, task order, output files, batch boundary, and task-specific constraints.
May add restrictions.
May not weaken repository rules or selection criteria.
Editing discipline
Make the smallest necessary change.
Edit only files named by the active task plan.
Do not rewrite unrelated candidate entries.
Do not reformat the repository without instruction.
Do not rename or move files unless the task explicitly requires it.
Do not add candidates unless instructed.
Do not commit, push, install dependencies, or change repository structure unless instructed.
Writing standard

Write repository content in English unless the task plan says otherwise.

Use concise, factual language.

Prefer:

specific area labels
evidence-led reasoning
short sentences
observable claims
explicit limitations

Avoid:

generic praise
inflated authority language
unsupported certainty
marketing language
“thought leader,” “world-class,” or similar empty labels
long biographies
repeated context already present elsewhere
Final task report

At the end of each task, report:

### Evidence gaps
